# Reference Image Selection for VEO 3.1

> How to select and configure reference images for VEO 3.1's reference image feature to improve character and environment consistency.

## Overview

VEO 3.1 supports up to 3 reference images per API call. These images guide the model to maintain visual consistency with established character appearances, outfits, and environments.

**Important:** This feature is VEO 3.1 only. VEO 2.0 does NOT support reference images.

## Compatibility

| Model | Reference Images | Max Images |
|-------|-----------------|------------|
| veo-3.0-generate-001 | Yes | 3 |
| veo-3.0-generate-001 | Yes | 3 |
| veo-2.0-generate-001 | No | 0 |

## Selection Priority

When a scene features a character, select reference images in this priority order:

### Priority 1: Character Front View
```
characters/{char-id}/styles/{active_style}/reference-front.png
```
The primary character reference. Always include when available. This establishes the character's face, hair, skin tone, and overall appearance.

### Priority 2: Outfit Full Body
```
characters/{char-id}/styles/{active_style}/outfits/{active_outfit}/full-body.png
```
Shows the character's current outfit in full body view. Include when the scene shows the character's body (medium shot or wider).

### Priority 3: Environment Wide Shot
```
characters/environments/{environment-id}/reference-wide.png
```
Establishes the setting. Include when the scene takes place in a specific, previously-defined environment.

## Configuration Format

```yaml
reference_images:
  - image: "characters/marcus/styles/realistic/reference-front.png"
    reference_type: "asset"
  - image: "characters/marcus/styles/realistic/outfits/business-suit/full-body.png"
    reference_type: "asset"
  - image: "characters/environments/hotel-lobby/reference-wide.png"
    reference_type: "asset"
```

### Rules

1. `reference_type` must always be `"asset"`
2. Maximum 3 images — if you have more candidates, use the priority order above
3. Image paths must resolve to existing files (generated by `generate-character-refs` workflow)
4. Images should match the target aspect ratio when possible (16:9 or 9:16)

## Selection Decision Tree

```
Scene has character?
├── Yes
│   ├── Character front view exists? → Add as reference #1
│   ├── Scene shows body (medium+ shot)?
│   │   └── Outfit full-body exists? → Add as reference #2
│   └── Scene has specific environment?
│       └── Environment reference exists? → Add as reference #3
└── No
    └── Scene has specific environment?
        └── Environment reference exists? → Add as reference #1
```

## Multi-Character Scenes

For scenes with multiple characters:
- Prioritize the primary/speaking character's references
- If 2 characters: use front view of each + environment (3 total)
- If 3+ characters: use front views of the 2 most prominent + environment
- Character tokens in the prompt compensate for characters without reference images

## When NOT to Use Reference Images

- **VEO 2.0 calls** — not supported
- **Abstract/motion graphics scenes** — no character or environment to reference
- **Text overlay scenes** — reference images may confuse the model
- **Scene type: screen-recording** — Playwright captures, no VEO involved

## Quality Requirements for Reference Images

Reference images should be:
- Generated by Nano Banana at 2K+ resolution
- Clean, well-lit, with neutral or matching background
- Consistent with the character's prompt tokens
- Saved in PNG format for maximum quality

## Validation

Before using reference images in a VEO call:
1. Verify the file exists at the specified path
2. Verify the model supports reference images (`features.reference_images == true`)
3. Verify no more than 3 images are included
4. Verify `reference_type` is set to `"asset"` for all images
