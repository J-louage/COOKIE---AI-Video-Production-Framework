# Merge Scenes — First/Last Frame Continuity Approach

## Purpose

This document provides a detailed, step-by-step guide for merging individually generated video clips into seamless scenes using FFmpeg's concat demuxer, with a focus on the first/last frame continuity technique that ensures visual coherence between adjacent clips.

---

## The Continuity Problem

VEO generates video clips in segments (typically 5–8 seconds each). When these clips are naively concatenated, visual discontinuities appear at the join points: sudden changes in lighting, character position, camera angle, or scene composition. The first/last frame continuity approach solves this by using the last frame of each clip as the starting reference for the next clip.

---

## Complete Workflow

### Step 1: Extract Last Frame from Clip A

After generating Clip A with VEO, extract its final frame. This frame becomes the starting point (first_frame) for Clip B.

```bash
ffmpeg -sseof -0.04 -i clip-A.mp4 -vframes 1 -q:v 2 last-frame-A.png
```

**Verification:**
```bash
# Confirm the frame was extracted and has reasonable dimensions
magick identify -format "%wx%h (%b)" last-frame-A.png
# Expected: 1920x1080 (similar to video resolution)
```

**Troubleshooting:**
- If the extracted frame is black, the video may have trailing black frames. Increase the offset: `-sseof -0.2` or `-sseof -0.5`.
- If the frame is garbled, the video may be corrupt at the tail. Try extracting from a slightly earlier point: `-sseof -1.0`.

### Step 2: Generate End-State Frame with Nano Banana

Generate the desired end state of Clip B using Nano Banana. This gives the VEO model both a starting point (last frame of Clip A) and an ending target, enabling interpolation between the two states.

```
Nano Banana prompt:
  [character:Marcus] [outfit:casual-blue-jacket]
  walking through park, reaching the bench, sitting down
  [camera:medium shot, eye level]
  [environment:city park, afternoon, dappled sunlight]
  [style:cinematic, warm tones]
```

Save the output as `target-frame-B.png`.

### Step 3: Generate Clip B with VEO Using Frame References

Submit the VEO generation request with:
- `first_frame`: `last-frame-A.png` (extracted from Clip A)
- `last_frame`: `target-frame-B.png` (generated by Nano Banana)
- `prompt`: Motion description for the clip

This tells VEO to create a video that starts from the exact visual state where Clip A ended and transitions to the target end state.

### Step 4: Repeat for Clip C, D, etc.

```bash
# Extract last frame from Clip B
ffmpeg -sseof -0.04 -i clip-B.mp4 -vframes 1 -q:v 2 last-frame-B.png

# Generate target end state for Clip C with Nano Banana
# ... (prompt for Clip C end state)

# Generate Clip C with VEO using:
#   first_frame = last-frame-B.png
#   last_frame = target-frame-C.png
```

### Step 5: Create the Clip List File

Once all clips are generated, create a text file listing them in order:

```bash
cat > clips.txt << 'EOF'
file 'clip-A.mp4'
file 'clip-B.mp4'
file 'clip-C.mp4'
EOF
```

**Important formatting rules:**
- One `file` entry per line.
- File paths must be quoted with single quotes.
- Paths are relative to the location of clips.txt (or use absolute paths).
- No blank lines or comments allowed in the file.
- Files must be listed in the correct playback order.

**For absolute paths:**
```bash
cat > clips.txt << EOF
file '/absolute/path/to/clip-A.mp4'
file '/absolute/path/to/clip-B.mp4'
file '/absolute/path/to/clip-C.mp4'
EOF
```

### Step 6: Merge with Concat Demuxer

```bash
ffmpeg -f concat -safe 0 -i clips.txt -c copy scene-merged.mp4
```

**Flags explained:**
- `-f concat` — Use the concat demuxer (not the concat filter, which is different).
- `-safe 0` — Allow unsafe file paths (required for absolute paths or paths with special characters).
- `-i clips.txt` — Input is the clip list file.
- `-c copy` — Copy streams without re-encoding. This is fast and lossless.

### Step 7: Verify the Merged Output

```bash
# Check duration
merged_duration=$(ffprobe -v error -show_entries format=duration -of csv=p=0 scene-merged.mp4)
echo "Merged duration: ${merged_duration}s"

# Calculate expected duration (sum of inputs)
expected=0
for clip in clip-A.mp4 clip-B.mp4 clip-C.mp4; do
  dur=$(ffprobe -v error -show_entries format=duration -of csv=p=0 "$clip")
  expected=$(echo "$expected + $dur" | bc)
done
echo "Expected duration: ${expected}s"

# Compare (allow 0.1s tolerance for frame-level rounding)
diff=$(echo "$merged_duration - $expected" | bc)
echo "Difference: ${diff}s"
```

### Step 8: Duration Validation

```bash
ffprobe -v error -show_entries format=duration -of csv=p=0 scene-merged.mp4
```

The merged duration should equal the sum of all input clip durations within a tolerance of 0.1 seconds. If the difference is larger:
- A clip may have been skipped (check clips.txt for typos).
- A clip may be corrupt (re-inspect individual clips).
- There may be a timestamp discontinuity (try re-encoding instead of `-c copy`).

## When `-c copy` Works

The copy codec mode (`-c copy`) works when all input clips share:

| Property | Must Match | How to Check |
|----------|-----------|-------------|
| Video codec | Same codec (e.g., all H.264) | `ffprobe -v error -select_streams v:0 -show_entries stream=codec_name -of csv=p=0 {clip}` |
| Resolution | Same width and height | `ffprobe -v error -select_streams v:0 -show_entries stream=width,height -of csv=p=0 {clip}` |
| Frame rate | Same fps | `ffprobe -v error -select_streams v:0 -show_entries stream=r_frame_rate -of csv=p=0 {clip}` |
| Pixel format | Same pixel format | `ffprobe -v error -select_streams v:0 -show_entries stream=pix_fmt -of csv=p=0 {clip}` |
| Audio codec | Same codec (if audio present) | `ffprobe -v error -select_streams a:0 -show_entries stream=codec_name -of csv=p=0 {clip}` |
| Audio sample rate | Same sample rate | `ffprobe -v error -select_streams a:0 -show_entries stream=sample_rate -of csv=p=0 {clip}` |

**VEO output consistency:** VEO typically generates all clips with the same codec, resolution, and frame rate, so `-c copy` should work in most cases. Always verify with a quick `ffprobe` check before merging.

---

## When Re-Encoding Is Needed

If any of the properties above differ between clips, you must re-encode during concatenation:

```bash
ffmpeg -f concat -safe 0 -i clips.txt \
  -c:v libx264 -crf 23 -preset medium \
  -c:a aac -b:a 192k \
  -movflags +faststart \
  scene-merged.mp4
```

**Flags explained:**
- `-c:v libx264` — Encode video with H.264.
- `-crf 23` — Constant Rate Factor. Lower = better quality, larger file. Range: 0–51. Recommended: 18–23.
- `-preset medium` — Encoding speed/compression tradeoff. Options: ultrafast, superfast, veryfast, faster, fast, medium, slow, slower, veryslow. Slower = better compression.
- `-c:a aac` — Encode audio with AAC.
- `-b:a 192k` — Audio bitrate.
- `-movflags +faststart` — Move the moov atom to the beginning for web streaming.

**When to re-encode:**
- Different codecs across clips.
- Different resolutions (also apply scaling filter).
- Different frame rates (also apply fps filter).
- When you need to apply additional filters (color correction, overlays).
- When the `-c copy` output has playback issues (seeking problems, A/V desync).

## Full Worked Example: 24-Second Scene

A 24-second scene split into 3 clips of approximately 8 seconds each.

### Scene Setup

```yaml
scene_id: scene-03
duration_target: 24s
clips:
  - id: clip-A
    duration: 8s
    action: "Marcus walks into the coffee shop, door swings open"
  - id: clip-B
    duration: 8s
    action: "Marcus approaches the counter, looks at the menu board"
  - id: clip-C
    duration: 8s
    action: "Marcus orders, barista smiles and begins making the drink"
```

### Step-by-Step Execution

```bash
# === CLIP A ===
# Generate Clip A with VEO (first clip in scene, no first_frame reference)
# VEO generates: scene-03/clip-A.mp4 (8.04s, 1920x1080, H.264, 25fps)

# Inspect Clip A
ffprobe -v error -show_entries format=duration -of csv=p=0 scene-03/clip-A.mp4
# Output: 8.040000

ffprobe -v error -select_streams v:0 -show_entries stream=width,height,codec_name,r_frame_rate -of csv=p=0 scene-03/clip-A.mp4
# Output: 1920,1080,h264,25/1

# Extract last frame from Clip A
ffmpeg -sseof -0.04 -i scene-03/clip-A.mp4 -vframes 1 -q:v 2 scene-03/last-frame-A.png

# === CLIP B ===
# Generate target end state for Clip B with Nano Banana
# Nano Banana → scene-03/target-frame-B.png

# Generate Clip B with VEO
#   first_frame = scene-03/last-frame-A.png
#   last_frame = scene-03/target-frame-B.png
#   prompt = "Marcus approaches counter, looks up at menu board"
# VEO generates: scene-03/clip-B.mp4 (8.00s)

# Inspect Clip B
ffprobe -v error -show_entries format=duration -of csv=p=0 scene-03/clip-B.mp4
# Output: 8.000000

# Extract last frame from Clip B
ffmpeg -sseof -0.04 -i scene-03/clip-B.mp4 -vframes 1 -q:v 2 scene-03/last-frame-B.png

# === CLIP C ===
# Generate target end state for Clip C with Nano Banana
# Nano Banana → scene-03/target-frame-C.png

# Generate Clip C with VEO
#   first_frame = scene-03/last-frame-B.png
#   last_frame = scene-03/target-frame-C.png
#   prompt = "Marcus orders coffee, barista smiles and starts making drink"
# VEO generates: scene-03/clip-C.mp4 (8.08s)

# === MERGE ===
# Create clip list
cat > scene-03/clips.txt << 'EOF'
file 'clip-A.mp4'
file 'clip-B.mp4'
file 'clip-C.mp4'
EOF

# Merge (from the scene-03 directory)
cd scene-03
ffmpeg -f concat -safe 0 -i clips.txt -c copy scene-03-merged.mp4

# === VERIFY ===
# Check merged duration
ffprobe -v error -show_entries format=duration -of csv=p=0 scene-03-merged.mp4
# Output: 24.120000

# Expected: 8.04 + 8.00 + 8.08 = 24.12 ✓

# Check resolution and codec consistency
ffprobe -v error -select_streams v:0 -show_entries stream=width,height,codec_name -of csv=p=0 scene-03-merged.mp4
# Output: 1920,1080,h264 ✓

# Play a quick preview to verify join points
# (Manual step: review at 8.04s and 16.04s marks)
```

### Final Output

```
scene-03/
  clip-A.mp4              # 8.04s - Marcus enters coffee shop
  clip-B.mp4              # 8.00s - Marcus approaches counter
  clip-C.mp4              # 8.08s - Marcus orders
  last-frame-A.png        # Continuity reference
  last-frame-B.png        # Continuity reference
  target-frame-B.png      # Nano Banana end-state
  target-frame-C.png      # Nano Banana end-state
  clips.txt               # Concat demuxer input
  scene-03-merged.mp4     # Final merged scene (24.12s)
```

## Multi-Scene Episode Assembly

After individual scenes are merged, assemble them into a full episode:

```bash
# Create episode clip list
cat > episode-clips.txt << 'EOF'
file 'scene-01/scene-01-merged.mp4'
file 'scene-02/scene-02-merged.mp4'
file 'scene-03/scene-03-merged.mp4'
file 'scene-04/scene-04-merged.mp4'
EOF

# Merge all scenes into episode
ffmpeg -f concat -safe 0 -i episode-clips.txt -c copy episode-01.mp4

# Verify total duration
ffprobe -v error -show_entries format=duration -of csv=p=0 episode-01.mp4
```

**Note:** Scene-to-scene transitions (fades, cuts) are Phase 2 features. In Phase 1, scenes are joined with hard cuts.

## Troubleshooting

| Issue | Diagnostic | Fix |
|-------|-----------|-----|
| Black flash at join point | Extract frames around the join and inspect | Re-extract last frame with larger offset. Re-generate the clip. |
| Audio pop/click at join | Listen to the join at high volume | Add a 10ms crossfade in post (Phase 2) or re-generate the clip |
| Duration mismatch | Compare merged duration vs sum of inputs | Check for dropped frames. Try re-encoding instead of copy. |
| "Discarding invalid edit list" warning | Container metadata issue | Safe to ignore in most cases. Re-encode if playback is affected. |
| Merged file will not seek properly | Missing keyframes at clip boundaries | Re-encode with `-c:v libx264 -g 50` to force keyframe interval |
